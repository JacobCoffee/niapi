Pre-configured log config for application deps.

While we use structlog for internal app logging, we still want to ensure that logs emitted by any of our dependencies are handled in a non- blocking manner.

app.lib.log.__init__.configure(processors: Sequence[Processor]) → None[source]

Call to configure structlog on app startup.

The calls to get_logger() in controller.py to the logger that is eventually called after this configurator function has been called. Therefore, nothing should try to log via structlog before this is called.
